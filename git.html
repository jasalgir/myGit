C O N F I G U R A C I O N

VER CONFIGURACION

$ git config --global --list🆗 // para ver la configuracion global de git
$ git config --global user.name "nombre de usuario"🆗 // cambia o añade nombre de usuario
$ git config --global [variable git] "valor que quieres guardar"🆗 // para añadir o cambiar valores

CREAR UN LOG ALIAS:

$ git config --global alias.superlog "log --graph --abbrev-commit --decorate --date=relative --format=format:'%C(bold blue)%h%C(reset) - %C(bold green)(%ar)%C(reset) %C(white)%s%C(reset) %C(dim white)- %an%C(reset)%C(bold yellow)%d%C(reset)' --all"🆗
$ git superlog🆗 // Ejecuta el alias creado anteriormente.

LOG MEJORADO:

$ git log --graph --abbrev-commit --decorate --date=relative --format=format:'%C(bold blue)%h%C(reset) - %C(bold green)(%ar)%C(reset) %C(white)%s%C(reset) %C(dim white)- %an%C(reset)%C(bold yellow)%d%C(reset)' --all🆗

CONEXION CON GITHUB

$ ssh-keygen -t rsa -b 4096 -C "mail de conexion a github"🆗 // para crear las claves que permitirán la comunicación.
$ git remote add origin [dirección que nos da github del repositorio]🆗 // repositorio debe haber sido creado con anterioridad. 'origin' es el nombre que Github le da al repositorio.
$ git push origin master🆗 || para hacer commit al repositorio remoto.

G E N E R A L

INICIAR GIT

$ git init🆗 /// desde el directorio del repositorio.

AÑADIR AL COMMIT:

$ git add "lo que necesitas añadir"🆗
$ git add -A🆗 // añade todo al commit

CLONAR REPOSITORIO

$ git clone [https://direccion que nos sumnistra el lugar donde se ubica el repo.]🆗

LISTADO DE COMMITS

$ git shortlog🆗 // extrae información de los commits
$ git log🆗
$ git [alias creado previamente]🆗 // ejemplo: $ git superlog //

ESTADO DE LOS COMMITS

$ git status🆗

R A M A S

CREAR RAMA:

$ git branch "nombre rama"🆗
$ git checkout -b "nombre rama"🆗

CAMBIAR DE RAMA:

$ git checkout master🆗 // para cambiar a la rama "master"
$ git checkout "nombre rama"🆗
$ git checkout [nombre de TAG]🆗

FUSIONAR RAMAS (desde master o rama padre):

$ git merge "rama a fusionar"🆗 // fusionaremos desde la rama donde queremos absorver

- entraremos en una pantalla en la cual pulsaremos "o" para insertar una descripción de la fusión.
Con "esc" nos saldremos del modo edición.
Con ":x" grabaremos y finalizará la fusión.

REBASE RAMAS (desde una rama hija forzamos la unión de las ramas, añadiendo todo lo de la rama "master", manteniendose ambas ramas. Tras esto, tocará fusionar ramas.):

$ git rebase "rama a incluir"🆗

BORRAR RAMA:

$ git branch -d "rama"🆗

MODIFICAR ULTIMO COMMIT:

$ git commit -am "mensaje" --amend🆗 // "-am" incluye el git add, "--amend" modifica último commit.

T A G S

CREAR TAGS A LOS COMMITS:

$ git tag -a [version] -m [mensaje]🆗
$ git tag [version]🆗
$ git tag -a [version] [id de commit]🆗 // si entrara en modo 'edicion' para salir del modo ESC y despues ':x' para salir y grabar

CONSULTAR TAG

$ git tag🆗

CONSULTAR POR EL NOMBRE DEL TAG

$ git show [nombre del tag ejem: 'v1.0']🆗
$ git tag -l "[version a buscar]"🆗 // busca la version indicada.
